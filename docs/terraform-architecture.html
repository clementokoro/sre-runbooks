Module Dependency Flow — Terraform Figures Out Order Automatically
1️⃣ VPC Module
no dependencies
runs FIRST
outputs:
vpc_id · subnet IDs
+
1️⃣ IAM Module
no dependencies
runs PARALLEL with VPC
outputs:
instance_profile_name
→
2️⃣ Security Groups
needs vpc_id from VPC
waits for VPC to finish
outputs:
k8s_security_group_id
→
3️⃣ EC2 Module
needs VPC + SG + IAM
waits for ALL THREE
builds:
mgmt · master
worker1 · worker2
⚡ You NEVER specify the build order manually. Terraform reads the output→input references and resolves the order itself. Modules with no shared dependencies (VPC + IAM) run in parallel automatically.

🔐 Why workers have IAM profile, master does not:
The LBCAWS Load Balancer Controller — runs in Kubernetes, creates ALBs automatically from Ingress objects runs on workers. It needs AWS API permissions to create and manage ALBsApplication Load Balancers — AWS managed load balancers that route HTTP/HTTPS traffic. Master + mgmt make no AWS API calls. Giving them the profile would violate least privilege.
